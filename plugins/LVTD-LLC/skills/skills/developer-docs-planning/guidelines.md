# Developer Docs Planning Guidelines

Load the minimum files needed for the task.

## By Task

| What you're doing | Load these files |
|-------------------|------------------|
| Creating a full documentation plan | `references/core/knowledge.md`, `workflows/create-doc-plan.md` |
| Deciding whether, when, or how to document something | `references/core/knowledge.md`, `workflows/make-doc-decision.md` |
| Researching a feature before writing | `references/core/knowledge.md`, `workflows/research-feature-docs.md` |
| Planning scenario-based docs | `references/core/knowledge.md`, `workflows/plan-scenario-docs.md` |
| Integrating docs with Agile or scrum work | `references/core/knowledge.md`, `workflows/integrate-docs-agile.md` |
| Choosing content types | `references/core/knowledge.md` |
| Planning API or reference docs | `references/core/knowledge.md` |
| Planning release notes or changelogs | `references/core/knowledge.md` |
| Reviewing a proposed doc outline | `references/core/knowledge.md` |

## By Problem

| If you notice... | Load these files |
|------------------|------------------|
| The plan says "write docs" with no doc types | `references/core/knowledge.md` |
| One doc tries to onboard, explain, reference, and troubleshoot | `references/core/knowledge.md` |
| The README is expected to do all documentation work | `references/core/knowledge.md` |
| No source of truth is assigned for API details | `workflows/create-doc-plan.md` |
| The team wants docs before the product questions are settled | `workflows/make-doc-decision.md` |
| Engineers describe implementation but not user consequences | `workflows/research-feature-docs.md` |
| A feature needs an end-to-end real-world example | `workflows/plan-scenario-docs.md` |
| Documentation is not visible in sprint planning or definition of done | `workflows/integrate-docs-agile.md` |
