# Frontend Math Foundations Knowledge

Core concepts and foundational understanding for frontend math foundations.

## Overview

Use this skill as the foundation layer for front-end math. It helps translate a visual or interaction problem into arithmetic, algebra, ratios, geometry, trigonometry, coordinates, or a CSS-vs-JavaScript ownership decision.

Source: Chapter 1: Web dev math fundamentals; see `../source-map.md`.

## Key Concepts

### Arithmetic expression

**Definition**: A combination of numbers, units, operators, and parentheses that yields a value.

### Algebra

**Definition**: Using symbols for unknown or variable values so a relationship can be solved or reused.

### Ratio

**Definition**: A relationship between quantities, common in aspect ratios, scale, and spacing systems.

### Linear equation

**Definition**: An equation with a constant rate of change, useful for interpolation and fluid values.

### Inequality

**Definition**: A bounded comparison such as min/max conditions and breakpoint ranges.

### Coordinate system

**Definition**: A way to locate points in space for layout, transforms, motion, and drawing.

## Terminology

| Term | Definition |
|------|------------|
| slope | Rate of change between two values. |

## Common Misconceptions

- **Myth**: The browser just guesses layout math.
  **Reality**: CSS and JavaScript apply deterministic formulas; unexpected results usually come from using the wrong reference value.
- **Myth**: More breakpoints or constants make design more precise.
  **Reality**: Good formulas often reduce breakpoints and make behavior more predictable between known sizes.
