# Book TOC Lab Rules

Use these rules when planning, auditing, or revising a useful nonfiction book before drafting.

## Core Rules

### 1. Solve One Reader Problem First

Choose a specific reader and a specific moment in their journey.

- Prefer "best for this person in this situation" over "useful to everyone."
- Define what the reader already knows so the book does not reteach everything.
- Exclude adjacent topics that slow the path to the promised outcome.

### 2. Make The Promise Legible

The reader should be able to infer the book's value from the title, subtitle, description, and TOC.

- Write the promise in one or two sentences.
- Test it in conversation.
- Listen for misinterpretations instead of explaining them away.

### 3. Use Scope As A Deletion Tool

Do not strengthen scope by adding more. Strengthen it by deleting work that does not help the chosen reader reach the chosen outcome.

- Move timely tutorials, tool-specific guidance, or tangents to companion resources.
- Keep prerequisites only when they are necessary for the promise.
- State who the book is not for.

### 4. Design For DEEP In Order

Evaluate the book in this order:

1. Desirable: readers want the promised result.
2. Effective: the book helps average readers get that result.
3. Engaging: value arrives fast enough to keep reading.
4. Polished: the book is professionally presented.

Do not spend drafting energy polishing a book whose desire or effectiveness is unproven.

### 5. Write The Recommendation Story

Before drafting, write the likely word-of-mouth event.

- What problem does the future reader mention?
- Who hears it?
- Why does this book come to mind?
- What proof makes the recommender confident?

If the story feels forced, revisit promise or reader profile.

### 6. Title TOC Sections By Takeaway

Use section titles that reveal what the reader gains.

- Replace topic labels with learning outcomes.
- Add subsections when a chapter contains several major takeaways.
- Save clever or literary naming until the structure has been validated.

## Guidelines

- Start with the smallest book that can fully deliver the promise.
- Put reader language into the promise and TOC when available.
- Use real or simulated reader conversations before committing to a draft.
- Treat confusion, boredom, and skepticism as design signals.

## Exceptions

- **Narrative nonfiction**: It may need more atmosphere or chronology, but the reader promise should still be legible.
- **Reference books**: A browsable taxonomy can be useful, but section names should still reveal practical value.
- **Advanced readers**: You can assume more context, but only if the reader profile makes those assumptions explicit.

## Quick Reference

| Rule | Summary |
|------|---------|
| Solve one problem | Narrow the reader, promise, and moment. |
| Legible promise | Let readers self-select quickly. |
| Delete by scope | Cut what does not serve the promise. |
| DEEP order | Desire and effectiveness come before polish. |
| Recommendation story | Make word of mouth concrete before drafting. |
| Takeaway TOC | Name outcomes, not author categories. |
