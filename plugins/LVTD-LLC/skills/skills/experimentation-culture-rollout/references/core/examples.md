# Experimentation Culture Rollout Examples

## Resistance Diagnosis

```markdown
Concern: "A/B testing will slow every release."
Likely root: Fear of process drag and missed deadlines.
Response: Start with significant user-facing changes only, use a one-page brief,
and publish time-to-launch metrics.
```

## Failed Experiment Reframe

```markdown
Result: The test variant did not improve the primary metric.
Learning: The assumed user pain was not strong enough to change behavior.
Decision: Do not launch broadly; run follow-up research with the segment that
showed mild positive movement.
```

**Why it works**:
- Treats the result as useful evidence.
- Explains what decision changed.
- Points to the next learning loop.

## First Rollout Slice

```markdown
Team: Personalization.
Surface: Homepage recommendation module.
Why this team: Real stakeholder debate, measurable engagement behavior, and
clear product decision.
Support: Analyst pairing, shared brief template, and readout review.
```

## Demand Generation Example

```markdown
Monthly experiment review:
- 3 tests launched.
- 2 readouts completed.
- 1 avoided launch due to guardrail regression.
- 1 follow-up test created from segment learning.
```

**Why it works**: It shows experimentation as a decision system, not a vanity
win-rate scoreboard.
