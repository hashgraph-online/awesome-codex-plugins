# Developer Docs Drafting Guidelines

Load the minimum files needed for the task.

## By Task

| What you're doing | Load these files |
|-------------------|------------------|
| Drafting a complete page | `references/core/knowledge.md`, `workflows/draft-developer-doc.md` |
| Drafting a scenario-based walkthrough | `references/core/knowledge.md`, `workflows/draft-developer-doc.md` |
| Improving titles, headings, or outline | `references/core/knowledge.md` |
| Writing procedures or task steps | `references/core/knowledge.md` |
| Making a draft more skimmable | `references/core/knowledge.md` |
| Creating a reusable doc template | `references/core/knowledge.md` |
| Writing self-contained SaaS topics | `references/core/knowledge.md`, `workflows/draft-developer-doc.md` |

## By Problem

| If you notice... | Load these files |
|------------------|------------------|
| The title describes a feature instead of a user goal | `references/core/knowledge.md` |
| The doc starts with background before the task | `references/core/knowledge.md` |
| Steps contain multiple actions or no expected result | `references/core/knowledge.md` |
| Callouts are overused or carry required information | `references/core/knowledge.md` |
| A scenario reads like a feature tour instead of a real user problem | `references/core/knowledge.md`, `workflows/draft-developer-doc.md` |
| A SaaS topic assumes the reader knows what the provider manages | `references/core/knowledge.md` |
