# LinkedIn Comment Writing Examples

## Extension Comment

```markdown
This is especially true when [specific context].

One thing I would add: [useful distinction].

Without that, teams often [negative consequence].
```

## Example Comment

```markdown
I saw this happen with [type of team/company].

They thought the issue was [surface issue], but the real constraint was
[deeper issue].

The fix was [short practical action].
```

## Respectful Disagreement

```markdown
I agree with the direction, but I would separate [case A] from [case B].

For [case A], this works because [reason].
For [case B], it can backfire because [reason].
```

## Better Question

```markdown
Useful framing.

How would you handle this when [specific edge case]?
That is where I usually see the trade-off become harder.
```

## Reply To A Comment On Your Own Post

```markdown
Good point, [Name].

The exception I have seen is [exception].
That is why I usually look for [signal] before applying this.
```

## Comment Audit Checklist

- Does it add something new?
- Is it specific to the post?
- Would the target audience respect it?
- Does it avoid hijacking the thread?
- Does it avoid fake engagement signals?
