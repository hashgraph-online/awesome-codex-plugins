# Build LinkedIn Content Backlog Workflow

Use this workflow when the user needs LinkedIn post ideas, a content calendar,
or thought-leadership topics.

## Steps

### Step 1: Define Territory

- [ ] Target audience.
- [ ] Topic the author should be known for.
- [ ] Offers, career goals, or business goals.
- [ ] Topics to avoid.

### Step 2: Mine Sources

- [ ] Audience questions.
- [ ] Industry news.
- [ ] Comments and replies.
- [ ] Search and hashtag conversations.
- [ ] Customer or reader language.
- [ ] Existing articles, talks, docs, podcasts, or research.

### Step 3: Convert To Hypotheses

- [ ] Write the reader problem.
- [ ] Write the author point of view.
- [ ] Choose a post job.
- [ ] Choose a format.
- [ ] Name the metric or signal to watch.

### Step 4: Prioritize

- [ ] Score audience value.
- [ ] Score author credibility.
- [ ] Score freshness.
- [ ] Score effort.
- [ ] Pick the first 3-5 ideas to draft.

## Exit Criteria

- [ ] Backlog has specific post angles, not generic topics.
- [ ] Each idea has a source and reason to exist.
- [ ] The first post can move directly into drafting.
