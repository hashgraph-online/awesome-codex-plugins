# Code Sample Documentation Guidelines

Load the minimum files needed for the task.

## By Task

| What you're doing | Load these files |
|-------------------|------------------|
| Reviewing a code sample | `references/core/knowledge.md`, `workflows/review-code-sample.md` |
| Writing an API example | `references/core/knowledge.md` |
| Checking request/response pairs | `references/core/knowledge.md` |
| Designing multi-language samples | `references/core/knowledge.md` |
| Evaluating autogenerated samples | `references/core/knowledge.md` |

## By Problem

| If you notice... | Load these files |
|------------------|------------------|
| The sample uses `foo`, fake IDs, or unclear placeholders | `references/core/knowledge.md` |
| Output does not match input or current API behavior | `workflows/review-code-sample.md` |
| A sample needs a long explanation to be understood | `references/core/knowledge.md` |
| Generated reference examples are technically correct but not useful | `references/core/knowledge.md` |
