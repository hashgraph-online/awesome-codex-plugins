# Docs Agile Integration Guidelines

Load the minimum files needed for the Agile documentation task.

## By Task

| What you're doing | Load these files |
|-------------------|------------------|
| Integrating docs work into sprint or release planning | `references/core/knowledge.md`, `workflows/integrate-docs-agile.md` |
| Making docs tasks visible on an Agile board | `references/core/knowledge.md`, `workflows/integrate-docs-agile.md` |
| Updating definition of done or release readiness for docs | `references/core/knowledge.md`, `workflows/define-docs-done.md` |
| Planning review timing with engineers, PM, QA, support, or field teams | `references/core/knowledge.md`, `workflows/integrate-docs-agile.md` |
| Handling documentation that spans multiple sprints or teams | `references/core/knowledge.md`, `workflows/integrate-docs-agile.md` |
| Improving docs process through retrospectives | `references/core/knowledge.md` |

## By Problem

| If you notice... | Load these files |
|------------------|------------------|
| Docs are discovered after code is done | `workflows/define-docs-done.md` |
| Docs work is not represented in stories, epics, or boards | `workflows/integrate-docs-agile.md` |
| Writers are invited only for final copy edits | `references/core/knowledge.md` |
| Review requests arrive too late for accurate docs | `workflows/integrate-docs-agile.md` |
| Definition of done ignores docs, release notes, or support readiness | `workflows/define-docs-done.md` |
| Scenario, troubleshooting, IA, or maintenance work does not fit a feature sprint | `references/core/knowledge.md` |

## File Index

| File | Purpose |
|------|---------|
