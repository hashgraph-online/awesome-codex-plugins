# Run Beta Iteration Workflow

Use this workflow to run one beta reading round for a practical nonfiction manuscript.

## When To Use

- A draft is coherent but not finished.
- The author needs real reader data before polishing.
- A previous beta round produced a revised manuscript.

## Prerequisites

- Manuscript or manuscript section.
- Target reader profile.
- Tool for live comments.

**Reference**: `references/core/rules.md`

---

## Workflow Steps

### Step 1: Set The Round Objective

**Goal**: Decide what this iteration is testing.

- [ ] State the manuscript stage.
- [ ] Choose the main learning goal: desirability, effectiveness, engagement, accuracy, or polish.
- [ ] Decide what feedback readers should ignore for now.

---

### Step 2: Prepare The Manuscript

**Goal**: Make feedback easy to give and integrate.

- [ ] Move the draft into a live commenting tool.
- [ ] Check permissions.
- [ ] Add reader instructions at the top.
- [ ] Mark any sections that are intentionally rough.

---

### Step 3: Recruit Readers

**Goal**: Get enough target readers for useful data.

- [ ] Define desired deeply engaged reader count.
- [ ] Build an invitation list larger than that count.
- [ ] Avoid using influential readers too early.
- [ ] Send a clear invitation with time expectations.

---

### Step 4: Monitor Signals

**Goal**: Capture visible and hidden feedback.

- [ ] Track who opens or comments.
- [ ] Track where comments cluster.
- [ ] Track where comments stop.
- [ ] Note strong value signals.
- [ ] Do not push readers through boredom.

---

### Step 5: Close The Round

**Goal**: Convert the round into revision work.

- [ ] Thank readers.
- [ ] Ask targeted follow-up questions where needed.
- [ ] Export or preserve comments.
- [ ] Move to `workflows/synthesize-feedback.md`.

## Exit Criteria

Task is complete when:

- [ ] Reader comments and drop-off points are preserved.
- [ ] Follow-ups have been sent where useful.
- [ ] The author is ready to synthesize before revising.
