# Synthesize Feedback Workflow

Use this workflow to turn beta reader comments into revision priorities.

## When To Use

- A beta reading round has ended.
- The manuscript has many comments and unclear next steps.
- Readers abandoned, disagreed, or signaled confusion.

## Prerequisites

- Preserved comments or exported feedback.
- Manuscript section map or TOC.

**Reference**: `references/core/examples.md`

---

## Workflow Steps

### Step 1: Build A Signal Table

**Goal**: Classify feedback before editing.

- [ ] Create rows for manuscript sections.
- [ ] Count or summarize comments by signal type.
- [ ] Mark comment drop-off points.
- [ ] Separate target-reader feedback from non-target-reader preferences.

Use this table:

```text
Section | Value | Confusion | Skepticism | Missing info | Boredom/drop-off | Accuracy | Revision idea
```

---

### Step 2: Preserve Value

**Goal**: Avoid cutting the parts that work.

- [ ] Identify examples, sections, or tones readers loved.
- [ ] Decide whether to move strong value earlier.
- [ ] Look for patterns that can be repeated elsewhere.

---

### Step 3: Fix Failure Modes

**Goal**: Choose manuscript changes by signal.

- [ ] For confusion: clarify, reorder, define, or add prerequisite.
- [ ] For skepticism: add proof, caveat, example, or soften claim.
- [ ] For boredom: cut, compress, or move value earlier.
- [ ] For inaccuracy: verify, correct, or ask an expert.
- [ ] For unnecessary drama: delete or de-escalate.

---

### Step 4: Rank Revisions

**Goal**: Work on the highest-leverage changes first.

- [ ] Prioritize problems affecting multiple readers.
- [ ] Prioritize earlier sections over later sections.
- [ ] Prioritize blockers over polish.
- [ ] Save nice-to-have wording changes for later.

---

### Step 5: Decide The Next Round

**Goal**: Know whether to beta read again or polish.

- [ ] If readers still fail to receive value, revise and run another beta round.
- [ ] If readers receive value but surface accuracy issues, do expert review.
- [ ] If readers reach the end and recommend it, move toward polish.

## Exit Criteria

Task is complete when:

- [ ] Feedback is classified.
- [ ] Revision priorities are ordered.
- [ ] The next manuscript milestone is clear.
