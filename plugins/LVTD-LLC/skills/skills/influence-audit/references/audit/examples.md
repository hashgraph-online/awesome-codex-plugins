# Influence Audit Examples

Examples are paraphrased applications of the principle, not book excerpts.

## Bad and Better Examples

### Fake urgency in checkout

Bad: A checkout page says only a few spots remain, but capacity is not constrained. Better: State the real enrollment cap or remove the claim; use a clear deadline only if operations require one.

### Staged popularity

Bad: A page implies thousands of users are joining today without data. Better: Use dated usage metrics, named customer segments, or no popularity claim.

### Authority overreach

Bad: An advisor logo implies product endorsement. Better: Explain the advisor role and avoid implying approval they did not give.

## Rewrite Pattern

1. Identify the active cue.
2. Remove unsupported claims or coercive framing.
3. Add the real evidence, constraint, or user benefit.
4. Make the next step clear but optional.

## Mini Review Example

**Input**: A message uses pressure but does not show evidence.

**Output**:
- Principle: Influence Audit
- Risk: unsupported cue may create a shortcut response.
- Rewrite: state the real reason, proof, or constraint; remove fake pressure.
