# Contrast and Reasons Rules

Rules for applying contrast and reasons without counterfeit evidence or coercive pressure.

## Core Rules

### 1. Use meaningful reasons

A reason should add decision-relevant information, not just include a reason word.

### 2. Compare like with like

Avoid sequencing that makes one option look better only because the comparison set is distorted.

### 3. Show total cost early

If add-ons are likely, present the total package or expected range before incremental add-ons.

### 4. Label anchors honestly

An anchor should reflect a real option, market price, or constraint, not a fake reference point.

## Guidelines

- Use contrast to help users see tradeoffs, not to bury cost.
- When presenting tiers, include best-fit guidance and who should not choose the premium tier.
- When asking for a favor, give the real reason and keep the request proportionate.

## Ethical Boundaries

- Use only honest evidence. Do not invent popularity, scarcity, credentials, endorsements, or social connection.
- Separate helping a good decision from pushing a shortcut response. If the cue is counterfeit, treat it as a red flag.
- When rewriting, preserve user agency: add context, alternatives, and enough time to decide when stakes are meaningful.

## Exceptions

- **Parody, critique, or analysis**: Describe manipulative tactics when the purpose is detection, education, or safer rewriting.
- **Real constraints**: Scarcity, authority, popularity, or reciprocity cues are acceptable when accurate, scoped, and relevant.
- **Tiny low-stakes choices**: Lighter disclosure may be fine, but never fabricate evidence.
