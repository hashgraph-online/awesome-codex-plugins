# Contrast and Reasons Examples

Examples are paraphrased applications of the principle, not book excerpts.

## Bad and Better Examples

### Pricing proposal

Bad: Show a very high fake package first so the target offer seems cheap. Better: show realistic tiers with scope, fit, and total cost.

### Add-on sequence

Bad: sell a base plan then reveal required add-ons one by one. Better: show the common full configuration upfront.

### Request

Bad: 'Can you do this because I need you to do this?' Better: state why it matters, deadline, and the smallest useful action.

## Rewrite Pattern

1. Identify the active cue.
2. Remove unsupported claims or coercive framing.
3. Add the real evidence, constraint, or user benefit.
4. Make the next step clear but optional.

## Mini Review Example

**Input**: A message uses pressure but does not show evidence.

**Output**:
- Principle: Contrast and Reasons
- Risk: unsupported cue may create a shortcut response.
- Rewrite: state the real reason, proof, or constraint; remove fake pressure.
