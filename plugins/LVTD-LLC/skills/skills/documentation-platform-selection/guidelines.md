# Documentation Platform Selection Guidelines

Load the minimum files needed for the platform decision.

## By Task

| What you're doing | Load these files |
|-------------------|------------------|
| Choosing a documentation platform or authoring tool | `references/core/knowledge.md`, `workflows/evaluate-doc-platform.md` |
| Comparing docs-as-code, CMS, wiki, static site, DITA, API reference, or help center options | `references/core/knowledge.md`, `workflows/evaluate-doc-platform.md` |
| Deciding whether tooling will solve a docs problem | `references/core/knowledge.md`, `workflows/evaluate-doc-platform.md` |
| Planning a pilot or proof of concept | `references/core/knowledge.md`, `workflows/plan-doc-migration-pilot.md` |
| Planning a docs migration | `references/core/knowledge.md`, `workflows/plan-doc-migration-pilot.md` |
| Building platform requirements | `references/core/knowledge.md` |

## By Problem

| If you notice... | Load these files |
|------------------|------------------|
| A team wants a new tool because docs are hard to find | `workflows/evaluate-doc-platform.md` |
| Current pain may come from ownership, IA, review, or stale content | `references/core/knowledge.md` |
| A migration plan omits redirects, metadata, search, or rollback | `workflows/plan-doc-migration-pilot.md` |
| Vendor features are being compared before requirements exist | `workflows/evaluate-doc-platform.md` |
| Content is trapped in a proprietary format | `references/core/knowledge.md` |
| No one will own customization, build, search, or support | `references/core/knowledge.md` |

## File Index

| File | Purpose |
|------|---------|
| `references/core/knowledge.md` | Platform concepts, requirement categories, rules, red flags, and patterns |
| `workflows/evaluate-doc-platform.md` | Workflow for comparing platform options |
| `workflows/plan-doc-migration-pilot.md` | Workflow for pilot and migration planning |

## Companion Skills

Use `documentation-information-architecture` when the main problem is navigation, content structure, or findability independent of tool choice. Use this skill when a platform, migration, authoring workflow, or tooling decision is central.
