# using-gc

Meta skill explaining how to run AgentOps on the Gas City (gc) substrate. gc is a guided out-of-session dependency — `ao` does NOT wrap it. Covers the gc primitives (City/Rig/Pack/Agent/Order/Formula/mayor/refinery), the AgentOps-on-GC workflow (gc init/start/rig add + the mayor-driven dispatch loop), cron exec Orders for maintenance, and the honest order-auto-dispatch gap (soc-5jwah).
